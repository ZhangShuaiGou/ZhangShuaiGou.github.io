Iterative BST Validation

Today’s problem is a classic problem, validate a BST.

Description:

Given the root node of a Binary Tree, write a method - validate_bst_itr to iteratively determine if it is a Binary Search Tree.

A BST must satisfy the following conditions :
* The left subtree of a node contains nodes with data < its data.
* The right subtree of a node contains nodes data > its data.
* A node’s left and right subtrees follow the above two conditions.

Examples:

There are several methods to figure it out. I used recursive approach.

The key point is the left->value is always less than the root->value and the right->value is always greater than root->value. Construct the recursive function based on this rule.

However, when I submitted this solution, I noticed that the problem was asking us to solve it via iteratively instead of recursively. Therefore I learned about the following solution, which via In-Order traversal. The steps or ideas can be found this link and this link.

Step 1: Create stack to store the nodes.

Step 2: Push the root node and the left child, and the left children of left child… I used root = root->left, however, it could be a pointer like cur_nd = root->left and cur_nd = cur_nd->left to make the code more readable.

Step 3: This push will stop at there is no left child among the left children of root node. Now, the top of the stack is the “root” node of the last “left node”. Then pop this node and make its right child as the new cur_nd and go back to step 2.

Step 4: So long, we only need to check whether the current node’s value is smaller than previous node’s. Because for the BST, the value of node will be displayed as ascend.

1-800-PROBLEM

Today I have reached to level 4 in Firecode. The first problem is quite challenging. I have no idea to figure it out, so I search it online for solution. And I leaned from the solution and write down my solution.

Description:

Given a string that represents the digits pressed on a classic cell phone keypad - return all possible letter combinations that the numbers could represent in an vector of strings. Check out the keypad and mapping below for reference.

Note:
a) You can assume that the input String contains only numbers between 2 and 9.
b) The order of the combinations in the output does not matter.

Firstly, I misunderstood the problem. I thought I need to pair every number with another number from the digits, so my first idea is apply nested for() loop to figure it. However, it is definitely wrong. Because then I realised that the question was not asking me to pair elements two by two, but to form as long a string as there were elements. So, the required nested for() loops as much as the elements number in digits, which is not predictable and impossible to program.

Then I struggled with it, I did not realize that to use recursive method till I searched online. I have found the solution and explanation of this problem in GeeksForGeeks. BTW, this website is amazing for learning programming.

The basic idea of this solution is replace the for() loop with recursions. Now I think about it, I should have actually thought of the recursive approach when I realised I needed multiple layers of loops.

In the recursive function, there are five parameters past-in: string tmp for store the combined strings. vector<string> &mp for the mapping of phone number to the alphabets, it contains two empty strings at front, so that for example the mp[2] will correspond exactly to the keypad 2, which mapping to “ABC”. int cur_idx for mark the current position of digits. string digits is from the question input. vector<string> &result to store the result, that is every finished tmp.

Firstly, it need to be determined that whether the cur_idx is beyond the length of digits. If ‘YES’, it means the current tmp is finished, it should be push into the result vector, then start the next round for the other combinations.

If ‘NO’, that means we need to keep going and finish this round. So we need to find the alphabets to be combined in this round. According to the mp, we can easily find the alphabets mapping to the current digit in digits.

Then here will be a for() loop to combine each single alphabet to the tmp. During this process, a new or longer tmp is created, however, our goal is the make the size of tmp equals to digits. So, in this loop, for every single alphabet, we need to call this function again to help us finish the task.

Now, the recursion is built and the problem solved.
